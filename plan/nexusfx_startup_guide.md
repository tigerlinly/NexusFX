# 🚀 NexusFX: Start-Up & Testing Guide

มาถึงจุดเริ่มต้นการทดสอบและการใช้งานจริงของแพลตฟอร์ม **NexusFX** แล้วครับ! 
ระบบทั้งหมดถูกประกอบร่างเรียบร้อยแล้ว ตั้งแต่หน้าบ้าน (Frontend - React/Vite) ไปจนถึงระบบรันบอทหลังบ้าน (Backend Background Workers) นี่คือคู่มือแบบทีละก้าว (Step-by-step) ที่จะช่วยให้คุณจำลองการทำงานของ "พอร์ตเทรดระดับองค์กรแบบอัตโนมัติ" ได้ด้วยตัวเองครับ

---

## 🟢 1. วิธีเข้าสู่แพลตฟอร์ม (Access & Login)

ระบบที่เราเพิ่ง Seed ข้อมูลตัวอย่างลงไป มีบัญชีพร้อมให้คุณใช้ Login เพื่อดูข้อมูลทั้งหมดได้ทันที

1. **เปิดเบราว์เซอร์** ไปที่ Frontend URL: `http://localhost:5173`
2. **เข้าสู่ระบบ** ด้วยข้อมูลจำลอง (Demo Data):
   - **รหัสผู้ใช้ (Username):** `demo`
   - **รหัสผ่าน (Password):** `demo1234`
3. ทันทีที่เข้าสู่ระบบ คุณจะเข้าสู่เขตหวงห้าม (Isolated Tenant Context) ซึ่งตอนนี้จะเห็นเพียงสรุปยอดเงินและข้อมูลการเทรดเฉพาะของบัญชี `demo` นี้เท่านั้น

## 🎨 2. สิ่งที่ควรทดสอบกวาดสายตา (UI & Dynamic Aesthetics)

เพื่อให้เห็นภาพสถาปัตยกรรม UI ที่คุณต้องการให้เป็น "Dynamic Web Design" ให้จับตาดูหรือเอาเมาส์ไปชี้ตามจุดต่างๆ เหล่านี้:

* **Dashboard สรุปยอด (Stat Grid):** เอาเมาส์ไปชี้กล่อง `ยอดคงเหลือ`, `กำไรวันนี้`, `Win Rate` คุณจะพบว่ากล่องจะกระเด้งลอยตัวขึ้นพร้อมมีแสง (Glow Box-Shadow) เปล่งประกายขึ้นมา
* **วิดเจ็ตเป้าหมาย (Daily Target Widget):** ขวามือจะมีหลอดเป้าหมายรายวัน ถ้าวันนี้ยอดกำไรของคุณทะลุเป้า ($100) ตัวกล่องจะมีการส่งสัญญาณแสงสีเขียวกระพริบเป็นจังหวะ (Pulse Animation) ทันที
* **ตารางประวัติรายวัน/รายสัปดาห์ (Reports):** เข้าไปที่เมนูโหมดรายงาน กดสลับแท็บ คุณจะเห็นการดึงข้อมูลรายสัปดาห์ในพริบตา และทดสอบกดปุ่ม `[Export]` เพื่อลองดาวน์โหลดไฟล์ CSV ผลการวิเคราะห์

## 📲 3. การทดสอบ Line Notify เชื่อม Backend

เราจะลองให้ระบบทักไปหาไลน์เรากันครับ! เพื่อยืนยันว่ากลไกแจ้งเตือนทำงาน

1. เข้าไปที่เมนู **ตั้งค่า (Settings)** บริเวณแถบซ้ายล่างสุด
2. เลื่อนลงมาหาช่องกรอก **Line Notify Token**
3. เอา Token ของคุณ (หากมี) ไปหยอดใส่ หากคุณไม่มี สามารถไปออก Token ได้ฟรีที่ [notify-bot.line.me](https://notify-bot.line.me/) 
4. กดปุ่ม **"ทดสอบการส่ง"** ด้านข้าง
5. ระบบ Backend จะยิงแจ้งเตือนผ่าน API และเด้งเข้ามือถือของคุณทันที: 
   > *"✅ การทดสอบการเชื่อมต่อ Line Notify สำเร็จ! ระบบ NexusFX ของคุณพร้อมส่งแจ้งเตือนการเทรดแล้ว 🚀"*
6. เมื่อทดสอบแล้ว อย่าลืมกดปุ่ม **บันทึก (Save Changes)** ด้านบนขวา เพื่อเก็บรหัสลงสู่ Database เวลาพอร์ตมีออเดอร์เด้งจริงๆ มันจะดึง Token จากหน้าจอนี้มาใช้งานครับ

---

## 🕵️ 4. ทดสอบความปลอดภัย (Groups & Risk Engine)

ระบบ **Automated Risk Engine** ล้างพอร์ตเมื่อขาดทุนหนัก ทำงานแบบ Background อยู่เบื้องหลัง ลองเข้ามาชมหน้า Group:
1. ไปที่เมนู **ทีม / จัดการกลุ่ม (Team / Groups)**
2. คุณจะเห็นกลุ่ม `NexusFX Alpha Team` ที่เพิ่งสร้างมาสดๆ ร้อนๆ 
3. จะเห็นว่ามีกฎ (Limits) คุมไว้ว่า `Max Drawdown = -500`
4. **หากอยากทดสอบของจริง:** หากในอนาคต `demo` เกิดขาดทุนเกิน $500 ภายในวันเดียว ระบบหลังบ้านจะบังคับปิดออเดอร์ของคุณทุกออเดอร์อัตโนมัติโดยที่คุณไม่ได้กดอะไรเลย และระงับบอทคุณจนกว่าจะถึงวันรุ่งขึ้น

## 🤖 5. การทดสอบยิงออเดอร์ผ่าน Webhook (Execution Engine)

นี่คือไฮไลท์ของระบบ! เราจะจำลองเหตุการณ์ที่มีการตั้ง Alert ทาง TradingView แล้วยิงออเดอร์มา

**ขั้นตอนการทดสอบด้วยตัวเองผ่าน Terminal (หรือ Postman):**
1. เปิด Command Prompt / Terminal อันใหม่
2. วางคำสั่ง `curl` ด้านล่างนี้เพื่อใช้ยิงสัญญาณ Trade เข้าไประบบ Webhook ของเรา:

```bash
curl -X POST http://localhost:4000/api/webhooks/bot/1/signal -H "Content-Type: application/json" -d "{\"action\": \"BUY\", \"symbol\": \"XAUUSD\", \"price\": 2350.50, \"lot_size\": 0.5, \"secret\": \"nexusfx_token_here\"}"
```

**สิ่งที่จะเกิดขึ้นเบื้องหลัง (Magic Pipeline):**
- **[Webhook Route]** ตรวจสอบรับ Payload สั่งให้บันทึกลงฐานข้อมูล `orders` 
- **[Execution Engine]** รันลูปทุก 2 วินาที พอเจอปุ๊บ มันก็จะดึง API ของกระดาน (Binance/MT5) ของคนนั้นมายิงคำสั่งแบบเสี้ยววินาที จากนั้นจะอัปเดตเป็น `FILLED`
- **[Line Notify]** ถ้าคุณใส่ Token ไว้ที่ Settings (จากข้อ 3) หลังจากวิ่งผ่าน Execution สำเร็จ คุณจะได้รับข้อความแจ้งเตือนทางไลน์อัปเดตออเดอร์ใหม่ทันทีว่า: `"✅ Order Executed Symbol: XAUUSD Side: BUY Volume: 0.5 Broker: Exness Main"`
