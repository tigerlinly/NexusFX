# คู่มือผู้ดูแลระบบ (Administrator Manual) - NexusFX Platform

ยินดีต้อนรับสู่ **NexusFX Trading Platform**! คู่มือฉบับนี้จะอธิบายขั้นตอนที่ **"ผู้ดูแลระบบ (Admin)"** ต้องทำให้ครบถ้วนตามลำดับ เพื่อให้แพลตฟอร์มพร้อมใช้งาน ทั้งระบบการจ่ายเงิน, ระบบส่งคำสั่งเทรดอัตโนมัติ, และการควบคุมความเสี่ยง

---

## สรุป 4 ขั้นตอนที่ต้องทำก่อนให้ลูกค้าเข้าใช้งาน
1. **ติดตั้ง API Keys หลัก** ของแพลตฟอร์ม
2. **ลงทะเบียนโบรกเกอร์** เพื่อให้ผู้ใช้สามารถเลือกเชื่อมต่อพอร์ตได้
3. **ควบคุมความเสี่ยงตลอดเวลา** (Global Risk)
4. **จัดการแพ็กเกจ (Subscriptions) และสมาชิก**

---

### ขั้นตอนที่ 1: ตั้งค่า API และช่องทางแจ้งเตือน (Settings)
เพื่อให้เซิร์ฟเวอร์สามารถส่งยอดเทรด หรือส่งข้อความเข้า LINE ได้ แอดมินต้องตั้งค่า **API Keys หลัก** ให้เสร็จเป็นอย่างแรก

1. เมนูด้านซ้าย เลือก **"ตั่งค่าระบบ (Settings)"**
2. ภายใต้แท็บ **"API Keys"** ให้แอดมินนำรหัสต่อไปนี้มากรอก:
   - **Binance API / Secret:** เพื่อให้ระบบหลังบ้าน (Execution Engine) สามารถโยนคำสั่งเข้า Binance ของฝั่งสภาพคล่องได้
   - **LINE Notify Token:** ถอดรหัสมาจาก LINE Notify ออกแบบให้การแจ้งเตือน Error หรือคนเติมเงิน วิ่งเข้ากลุ่มผู้ดูแล
3. กดปุ่ม **บันทึก (Save Changes)** บริเวณด้านขวาบน

> **⚠️ ข้อควรระวัง:** หากไม่ได้ตั้งค่าบรรทัดนี้ หรือใส่คำว่า "your_binance_key" (ไม่มีรหัสจริง) ระบบเทรดจะเข้าสู่โหมด **Simulation Mode (ระบบจำลอง)** แทนการเทรดเงินจริงโดยอัตโนมัติ!

---

### ขั้นตอนที่ 2: จัดเตรียมลิสต์โบรกเกอร์ (Broker Management)
ผู้ใช้งานจะไม่สามารถผูกพอร์ตของเขาได้ หากแอดมินยังไม่เตรียมตัวเลือกโบรกเกอร์เอาไว้ในระบบ (เช่น Exness, IC Markets เป็นต้น)

1. กลับไปที่หน้าจอแดชบอร์ด แอดมินจะต้องแน่ใจว่าได้ทำการเพิ่มโบรกเกอร์เข้าในฐานข้อมูลหลัก (ปัจจุบันระบบมีข้อมูลอัตโนมัติ 3 แห่งคือ Exness, IC Markets, Binance)
2. หากโบรกเกอร์มีปัญหา สามารถเข้าไปปรับ **Markup Fee** (ส่วนต่างค่าธรรมเนียมที่เราจะเก็บจากลูกค้า) สำหรับโบรกเกอร์แต่ละแห่งได้

---

### ขั้นตอนที่ 3: แผงควบคุมฉุกเฉิน (Admin Control Panel)
ระบบขับเคลื่อนด้วยบอทนับร้อยตัวพร้อมกัน แอดมินคือผู้ถือ "เบรกฉุกเฉิน" ทั้งหมด

1. ไปที่เมนู **"แดชบอร์ดแอดมิน (Admin)"**
2. ระบบจะวิเคราะห์ภาพรวมตลอดเวลา ว่ามี **% Win Rate โดยรวม**, **ยอดเงินหมุนเวียน (AUM)** และมีบอทที่พอร์ตกำลังแตกหรือติดลบหนัก (Drawdown สูง) หรือไม่
3. **กรณีเกิดข่าวเศรษฐกิจรุนแรง (Black Swan):** หน้าต่างนี้จะมีปุ่มพิเศษสีแดงกำกับไว้คือ **"Global Kill Switch"** 
   - ทันทีที่แอดมินกดปุ่ม ระบบจะส่งสัญญาณ `SIG_KILL` ผ่าน WebSockets ไประงับ *ทุกบอทและทุกบัญชีโดยอัตโนมัติ*

### ขั้นตอนที่ 4: ตรวจสอบและอนุมัติแพ็กเกจ (Billing Management)
รายได้แพลตฟอร์มเกิดจากการที่สมาชิกยอมจ่ายค่าเช่ารายเดือน (Subscriptions) แอดมินสามารถเปิดสิทธิ์ "องค์กร" หรือแก้ไของค์กรย่อยต่างๆ:

1. สมาชิกจะกดจ่ายเงินด้วยยอด USD โดยระบบแพลตฟอร์มจะ **หักเงินจากระบบ Wallet** ให้เองอัตโนมัติ (และรัน Audit Log ถาวร)
2. แอดมินสามารถตรจสอบ Log การจ่ายเงิน (Billing Statement) ทั้งหมดได้จากฐานข้อมูล และมอบสิทธิการเข้าถึงฟีเจอร์ Multi-Terminal กับ Enterprise Users
